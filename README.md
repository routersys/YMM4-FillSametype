# 同型塗りつぶし for YMM4
 
[![License](https://img.shields.io/badge/license-LGPL-blue.svg)](#)
[![.NET](https://img.shields.io/badge/.NET-10.0-purple.svg)](#)
[![Release](https://img.shields.io/github/v/release/routersys/YMM4-FillSametype.svg)](https://github.com/routersys/YMM4-FillSametype/releases)
 
YukkuriMovieMaker4（YMM4）上で動作する、**画像内の同形状領域を自動検出してブラシで塗りつぶす映像エフェクトプラグイン**です。
HLSL ピクセルシェーダーを Direct2D カスタムエフェクトとして実装しており、GPU 上でのリアルタイムレンダリングが可能です。
指定座標のピクセル色を基準とした色差フィルタリングと、log 極座標ヒストグラムによる形状特徴量を用いた類似形状マッチングにより、対象と同形の連結成分を一括で塗りつぶします。

## 概要
 
本プラグインは YMM4 の映像エフェクトとして動作します。画像上の対象座標（X・Y）を指定すると、その座標のピクセル色を種色として読み取り、前景マスクを生成します。前景内に存在する複数の連結成分のうち、種成分と同形のものをまとめてブラシで塗りつぶします。
 
Direct2D カスタムピクセルシェーダーエフェクト（`D2D1CustomShaderEffectBase`）として実装されており、コンパイル済み HLSL シェーダー（`.cso`）を利用します。形状の類似度判定には ComputeSharp による GPU シェーダーを用いた log 極座標ヒストグラムのコサイン類似度比較を使用します。

## インストール方法
 
1. [Releases](https://github.com/routersys/YMM4-FillSametype/releases/latest) ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4 が起動していないことを確認し、ダウンロードしたファイルを実行してインストールします。
3. YMM4 を起動し、タイムライン上のアイテムに映像エフェクトを追加します。
4. 映像エフェクトの種類として **「同型塗りつぶし」** を選択してください。

## 主な機能
 
### 1. 色差フィルタリングによる前景抽出
 
対象座標（X・Y）のピクセル色を種色として読み取り、**色許容範囲**パラメータに基づいて各ピクセルが前景か否かを判定します。判定は HLSL ピクセルシェーダー上で行われ、種色との RGBA 各チャンネル差のチェビシェフ距離が閾値以内であれば前景として扱われます。アルファ値が 0 のピクセルは前景から除外されます。
 
### 2. 連結成分ラベリング
 
前景マスクに対して CPU 側で Union-Find による連結成分ラベリングを行い、前景ピクセルを連結した塊ごとに分類します。連結性の判定は 8 方向で行われます。処理できる成分数の上限は 65,536 です。
 
### 3. log 極座標ヒストグラムによる形状マッチング
 
各連結成分について、重心を中心とした log 極座標ヒストグラムを特徴量として GPU 上で算出します。その後ヒストグラムを L2 正規化し、種成分と各成分のコサイン類似度を全角度シフトにわたって計算することで、回転不変な形状マッチングを行います。**同型許容範囲**パラメータで類似度の閾値を調整でき、閾値以上の成分をすべて塗りつぶし対象とします。
 
### 4. ブラシによる塗りつぶし
 
マッチした成分の領域に対して、YMM4 標準のブラシ機能を使って塗りつぶしを行います。ブラシの種類はプロパティパネルの「模様」グループから変更できます。
 
### 5. 境界ぼかし・不透明度・合成モード
 
塗りつぶしマスクにガウシアンブラーによる境界ぼかしを適用することで、塗りつぶし領域の輪郭を滑らかにできます。不透明度パラメータで塗りつぶし全体の透明度を調整でき、合成モードでは YMM4 が提供する各種ブレンドモードが使用可能です。
 
### 6. 領域反転・輝度保持・模様のみモード
 
- **領域反転**: マッチした成分以外の領域を塗りつぶし対象にします。
- **輝度保持**: 塗りつぶし後の画像に元画像の輝度を Luminosity ブレンドで合成し、色相・彩度のみを変更します。
- **模様のみ**: 元の画像を描画せず、塗りつぶし結果のみを出力します。

## アルゴリズム
 
### 前景マスク生成（HLSL シェーダー）
 
`FillSamegroundShader.hlsl` が各ピクセルに対して以下の処理を行います。
 
1. 入力テクスチャからピクセル色をサンプリングします。
2. 種色との RGBA 差を計算し、各チャンネル差のチェビシェフ距離を `tolerance` と比較します。
3. 距離が閾値以内であれば `float4(1, 1, 1, 1)`、超過であれば `float4(1, 1, 1, 0)` を返します。
4. `invertMask > 0.5` の場合はマスクを反転します（本エフェクトでは常に `0` で呼び出されます）。
```hlsl
float4 diff = abs(color - targetColor);
float  dist = max(max(diff.r, diff.g), max(diff.b, diff.a));
float  mask = (dist <= tolerance) ? 1.0 : 0.0;
return float4(1.0, 1.0, 1.0, mask);
```
 
前景ピクセルの判定はアルファチャンネルの書き込みのみで行い、RGB チャンネルは常に白（1.0）を出力します。このマスク出力は `ForegroundExtractor` がアルファチャンネルのみを読み取り、値が 0 でなければ前景（1）、0 であれば背景（0）として int 配列に格納します。`ForegroundExtractor` は AVX2、SSE2、スカラーの順に利用可能な命令セットへフォールバックします。
 
### 連結成分ラベリング（CPU）
 
`FillSametypePipeline.Label` メソッドが処理を担います。
 
1. 全ピクセルを Union-Find 構造で初期化します。
2. ラスタスキャン順に各前景ピクセルについて、左・上・左上・右上の 4 近傍との Union 処理を行います（8 方向連結）。
3. Find 処理ではパス二段縮約（path halving）を行います（`parent[x] = parent[parent[x]]`）。
4. 最終的に各成分に連番ラベルを割り当てます。重心計算で面積 0 割り算を避けるため、面積が 16 ピクセル未満の成分はマッチング対象から除外されます。
```
// Union-Find パス二段縮約
while (parent[x] != x) {
    parent[x] = parent[parent[x]];
    x = parent[x];
}
```
 
### 特徴量算出・マッチング（GPU）
 
ComputeSharp を用いた GPU シェーダー群が処理を担います。
 
#### `LogPolarHistogramShader`
 
各前景ピクセルについて、所属成分の重心からの距離・角度を計算し、角度方向 36 ビン × 半径方向 12 ビン（合計 432 次元）の 2D ヒストグラムに加算します。半径方向は `log(r) × logRadiusScale` による対数スケールを使用し、スケール変化への頑健性を高めています。重心からの距離が 1.0 未満のピクセルはビン計算を行いません（`log(r)` の発散回避のため）。`logRadiusScale` は画像対角線長の log 値から算出されます。
 
```
logRadiusScale = RadialBins / log(sqrt(width² + height²))
```
 
#### `NormalizeHistogramShader`
 
各成分のヒストグラムベクトルを L2 ノルムで除算し、浮動小数点の正規化済み特徴ベクトルに変換します。ノルムが 1e-6 未満の場合はゼロベクトルとして扱います。
 
#### `CorrelationMatchShader`
 
種成分と各成分の正規化済み特徴ベクトル間のコサイン類似度を、角度シフト 0〜35 の全 36 通りにわたって計算し、最大値を類似度スコアとします。角度シフトは角度ビンの巡回的なシフトとして実装されており、回転に依存しない形状マッチングを実現します。スコアが閾値以上の成分を一致とみなしてフラグを立てます。
 
類似度閾値は `1.0 - clamp(ShapeThreshold / 100, 0, 1)` として計算されます。**同型許容範囲**が 0 のとき閾値は 1.0（完全一致のみ）、100 のとき閾値は 0.0（すべて一致）となります。
 
#### `MaskShader`
 
ラベルバッファと一致フラグから最終マスクを生成します。一致した成分に属するピクセルを `-1`（0xFFFFFFFF）、それ以外を `0` とした int 配列を出力します。`invert` フラグに応じてマスクを反転します。

## 使い方ガイド
 
### 基本的な使い方
 
1. タイムライン上のアイテムを選択し、エフェクトを追加します。
2. 映像エフェクトの種類として **「同型塗りつぶし」** を選択します。
3. **X・Y** パラメータを調整し、塗りつぶしたい形状上の座標を指定します。ビューポート上のコントロールポイントをドラッグして位置を変更することもできます。
4. **色許容範囲**を調整して前景とみなすピクセルの範囲を設定します。
5. **同型許容範囲**を調整して、どの程度似た形状まで塗りつぶすかを設定します。値を大きくするほど類似度閾値が低下し、より多くの形状が一致とみなされます。
6. 「模様」グループのブラシ設定で塗りつぶしの色や模様を変更します。
 
### 色許容範囲の調整
 
**色許容範囲**は 0（種色との完全一致のみ）〜 255（全ピクセルを前景）の範囲で設定します。内部では `tolerance = ColorTolerance / 255.0` に変換され、RGBA 各チャンネルのチェビシェフ距離と比較されます。アルファ値が 0 のピクセルは常に前景から除外されます。
 
### 同型許容範囲の調整
 
**同型許容範囲**は 0〜100 のスケールで、`similarity_threshold = 1.0 - ShapeThreshold / 100.0` として類似度閾値に変換されます。値が 0 のとき閾値は 1.0（コサイン類似度が完全一致の成分のみ選択）、値が 100 のとき閾値は 0.0（すべての成分を選択）となります。
 
### 領域反転の利用
 
**領域反転**を ON にすると、一致した成分以外の領域が塗りつぶし対象になります。特定の形状だけを残して周囲を塗りつぶす用途に活用できます。
 
### 輝度保持の利用
 
**輝度保持**を ON にすると、塗りつぶし色の色相・彩度を適用しながら元画像の輝度を Luminosity ブレンドで保持します。元の陰影感を残したまま色を変更したい場合に有効です。
 
### キーフレームアニメーション
 
X・Y・不透明度・境界ぼかし・色許容範囲・同型許容範囲はすべてキーフレームアニメーションに対応しています。タイムライン上でキーフレームを設定することで、塗りつぶし対象の座標や範囲を時間とともに変化させることができます。

## 注意事項
 
- **AviUtl 非対応**: HLSL ピクセルシェーダーおよび Direct2D カスタムエフェクトを使用しているため、AviUtl 形式（`.exo`）への出力には対応していません。
- **成分数の上限**: 処理できる連結成分の上限は 65,536 です。これを超えた成分はラベリングの対象外となります。
- **最小面積フィルタ**: 面積が 16 ピクセル未満の連結成分はマッチング対象から除外されます。
- **種色座標が前景外の場合**: X・Y で指定した座標のピクセルが前景マスク上で前景と判定されない場合、マスクは生成されずエフェクトは適用されません。
- **GPU の利用**: 特徴量算出・マッチングには ComputeSharp を通じてデフォルト GPU デバイスが使用されます。GPU が利用できない環境では動作しない場合があります。
- **本プラグインを使用する前に、YMM4 プロジェクトファイルのバックアップを作成することを推奨します。**

# 同型塗りつぶし (FillSametype) for YMM4

YukkuriMovieMaker4（YMM4）上で動作する、**画像内の同形状領域を自動検出してブラシで塗りつぶす映像エフェクトプラグイン**です。
HLSL ピクセルシェーダーを Direct2D カスタムエフェクトとして実装しており、GPU 上でのリアルタイムレンダリングが可能です。
指定座標のピクセル色を基準にした色許容範囲フィルタリングと、ログ極座標ヒストグラムによる形状特徴量を用いた類似形状マッチングにより、対象と同形の領域を一括で塗りつぶします。

## 概要
 
本プラグインは YMM4 の映像エフェクトとして動作します。画像上の対象座標（X・Y）を指定することで、その座標のピクセル色を基準にした前景マスクを生成し、前景内に存在する複数の連結成分（形状）のうち、対象と同形のものをまとめてブラシで塗りつぶします。
 
内部的には Direct2D カスタムピクセルシェーダーエフェクト（`D2D1CustomShaderEffectBase`）として実装されており、HLSL コンパイル済みシェーダー（`.cso`）を利用します。形状の類似度判定には GPU 上の ComputeSharp シェーダーを用いたログ極座標ヒストグラムのコサイン類似度比較を使用します。

## インストール方法
 
1. [Releases](https://github.com/routersys/YMM4-FillSametype/releases/latest) ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4 が起動していないことを確認し、ダウンロードしたファイルを実行（ダブルクリック）してインストールします。
3. YMM4 を起動し、タイムライン上のアイテムに映像エフェクトを追加します。
4. 映像エフェクトの種類として **「同型塗りつぶし」** を選択してください。

## 主な機能
 
### 1. 色許容範囲による前景抽出
 
対象座標（X・Y）のピクセル色を種色として読み取り、**色許容範囲**パラメータに基づいて各ピクセルが前景か否かを判定します。判定は HLSL ピクセルシェーダー上で行われ、RGBA 各チャンネルの差の最大値が閾値以内であれば前景として扱われます。アルファ値が 0 のピクセルは前景から除外されます。
 
### 2. 連結成分ラベリング
 
前景マスクに対して CPU 側で **Union-Find（素集合データ構造）** による連結成分ラベリングを行い、画像内の前景ピクセルを互いに繋がった塊（成分）ごとに分類します。連結性の判定は 8 方向（水平・垂直・斜め）で行われます。処理できる成分数の上限は **65536** です。
 
### 3. ログ極座標ヒストグラムによる形状マッチング
 
各連結成分について、重心を中心とした**ログ極座標ヒストグラム**を特徴量として GPU 上で算出します（角度方向 36 ビン、半径方向 12 ビン）。その後、ヒストグラムを L2 正規化し、対象成分と各成分のコサイン類似度を全回転シフトにわたって計算（回転不変マッチング）します。**同型許容範囲**パラメータによって類似度の閾値を調整でき、閾値以上の成分をすべて塗りつぶし対象とします。
 
### 4. ブラシによる塗りつぶし
 
マッチした成分の領域に対して、YMM4 標準の**ブラシ**機能（単色・グラデーション・パターン等）を使って塗りつぶしを行います。ブラシの種類はプロパティパネルの「模様」グループから変更できます。
 
### 5. 境界ぼかし・不透明度・合成モード
 
塗りつぶしマスクに対してガウシアンブラーによる**境界ぼかし**を適用することで、塗りつぶし領域の輪郭を滑らかにできます。また**不透明度**パラメータで塗りつぶし全体の透明度を調整できます。**合成モード**では YMM4 が提供する各種ブレンドモードが使用可能です。
 
### 6. 領域反転・輝度保持・模様のみモード
 
- **領域反転**: マッチした成分以外の領域を塗りつぶし対象に反転します。
- **輝度保持**: 塗りつぶし後の画像に対して元画像の輝度を合成し、色相・彩度のみを変更して輝度を保持します。
- **模様のみ**: 元の画像を描画せず、塗りつぶしのみを出力します。

## アルゴリズム
 
### 前景マスク生成（HLSLシェーダー）
 
`FillSamegroundShader.hlsl` が各ピクセルに対して以下の処理を行います。
 
1. 入力テクスチャからピクセル色をサンプリングします。
2. 種色（`targetColor`）との RGBA 差を計算し、各チャンネル差の最大値（チェビシェフ距離）を `tolerance` と比較します。
3. 距離が閾値以内であれば `float4(1,1,1,1)`（白・不透明）、超過であれば `float4(1,1,1,0)`（白・透明）を返します。
4. `invertMask > 0.5` の場合はマスクを反転します（本エフェクトでは常に `0` で呼び出されます）。
```hlsl
float4 diff = abs(color - targetColor);
float  dist = max(max(diff.r, diff.g), max(diff.b, diff.a));
float  mask = (dist <= tolerance) ? 1.0 : 0.0;
return float4(1.0, 1.0, 1.0, mask);
```
 
### 連結成分ラベリング（CPU）
 
`FillSametypePipeline.cs` の `Label` メソッドが処理を担います。
 
1. 全ピクセルを Union-Find 構造で初期化します。
2. ラスタスキャン順（左→右、上→下）に各前景ピクセルについて、左・上・左上・右上の 4 近傍との Union 処理を行います（8 方向連結）。
3. ルートノードへの経路圧縮（パス圧縮）および rank によるマージを行います。
4. 最終的に各成分に連番ラベルを割り当てます。面積が `MinimumComponentArea`（16 ピクセル）未満の成分はマッチング対象から除外されます。
### 特徴量算出・マッチング（GPU）
 
ComputeSharp を用いた GPU シェーダー群が処理を担います。
 
**`LogPolarHistogramShader`**
 
各前景ピクセルについて、所属成分の重心からの距離・角度を計算し、角度ビン（36 分割）× 半径ビン（12 分割）の 2D ヒストグラムに加算します。半径方向は対数スケール（`log(r) × logRadiusScale`）を使用し、スケール不変性を高めています。
 
**`NormalizeHistogramShader`**
 
各成分のヒストグラムベクトルを L2 ノルムで正規化し、浮動小数点特徴ベクトルに変換します。
 
**`CorrelationMatchShader`**
 
種成分と各成分の正規化済み特徴ベクトル間のコサイン類似度を、全角度シフト（36 通り）にわたって計算し、最大値を類似度スコアとします。これにより回転に依存しない形状マッチングを実現します。スコアが閾値以上の成分を「一致」としてフラグを立てます。
 
**`MaskShader`**
 
ラベルバッファと一致フラグから最終マスクを生成します。`invert` フラグに応じてマスクを反転します。

## 使い方ガイド
 
### 基本的な使い方
 
1. タイムライン上のアイテムを選択し、エフェクトを追加します。
2. 映像エフェクトの種類として **「同型塗りつぶし」** を選択します。
3. **X・Y** パラメータを調整し、塗りつぶしたい形状上の座標を指定します。ビューポート上のコントロールポイントをドラッグして位置を変更することもできます。
4. **色許容範囲**を調整して前景とみなすピクセルの範囲を設定します。
5. **同型許容範囲**を調整して、どの程度似た形状まで塗りつぶすかを設定します。値を大きくするほど多くの形状が一致とみなされます。
6. 「模様」グループのブラシ設定で塗りつぶしの色や模様を変更します。
### 色許容範囲の調整
 
**色許容範囲**は 0（完全一致）〜 255（全ピクセル前景）の範囲で設定します。値が大きいほど種色に近い色のピクセルを多く前景として抽出します。アルファ値が 0 のピクセル（完全透明）は常に前景から除外されます。
 
### 同型許容範囲の調整
 
**同型許容範囲**は形状の類似度閾値を 0〜100 のスケールで指定します。値が大きいほど閾値が緩くなり、より多くの異なる形状が「同型」とみなされます。値が小さいほど、対象の形状に厳密に近い成分のみが塗りつぶされます。
 
### 領域反転の利用
 
**領域反転**を ON にすると、一致した成分**以外**の領域が塗りつぶし対象になります。特定の形状だけを残して周囲を塗りつぶす用途などに活用できます。
 
### 輝度保持の利用
 
**輝度保持**を ON にすると、塗りつぶし色の色相・彩度を適用しながら元画像の輝度を保持します。元の陰影感を残したまま色を変更したい場合に有効です。
 
### キーフレームアニメーション
 
**X・Y・不透明度・境界ぼかし・色許容範囲・同型許容範囲**はすべてキーフレームアニメーションに対応しています。タイムライン上でキーフレームを設定することで、塗りつぶし対象の座標や範囲を時間とともに変化させることができます。

## 注意事項
 
- **AviUtl 非対応**: 本プラグインは HLSL ピクセルシェーダーおよび Direct2D カスタムエフェクトを使用しているため、AviUtl 形式（`.exo`）への出力には対応していません。
- **成分数の上限**: 処理できる連結成分の上限は **65536** です。これを超えた成分はラベリングの対象外となります。
- **最小面積フィルタ**: 面積が **16 ピクセル未満**の連結成分はマッチング対象から除外されます。
- **種色座標が前景外の場合**: X・Y で指定した座標のピクセルが前景マスク上で前景と判定されない場合、マスクは生成されずエフェクトは適用されません。
- **ComputeSharp の GPU 利用**: 特徴量算出・マッチングには ComputeSharp を通じてデフォルト GPU デバイスが使用されます。GPU が利用できない環境では動作しない場合があります。
- **本プラグインを使用する前に、YMM4 プロジェクトファイルのバックアップを作成することを推奨します。**
